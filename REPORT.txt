PART 1 — THEORY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1  PROBLEM WITH RIGID-REFERENCE WER
──────────────────────────────────────
Standard WER is defined as:

    WER = (S + D + I) / N

where  S = substitutions, D = deletions, I = insertions,
       N = number of words in the reference.

This penalises a model whenever its output differs from the reference —
even when the difference is:
  (a) Punctuation noise        e.g. "feedback." vs "feedback"
  (b) Script variant           e.g. "sir" vs "सर", "feedback" vs "फ़ीडबैक"
  (c) Spelling variant         e.g. "हाँ" vs "हां", "सुविधाएं" vs "सुविधाएँ"
  (d) Compound split/merge     e.g. "खेतीबाड़ी" vs "खेती बाड़ी"
  (e) Human reference error    e.g. "नाथ से" (Model H) vs "नाते" (correct)
  (f) Word-form variant        e.g. "वो" vs "वह", "की" vs "कि"
  (g) Number form              e.g. "चौदह" vs "14"
  (h) Filler word presence     e.g. "हम्म" insertion or omission

In the provided dataset, the human reference contains demonstrable errors
(e.g., segment ch_696260: Human writes "हम्म बहुत..." while Model H drops
"हम्म"; segment ch_687596: Model H writes "नाथ से" where everyone else,
including the cleaner models, agrees on "नाते").

1.2  ALIGNMENT UNIT CHOICE — WORD
──────────────────────────────────
Three options: character, word, subword (BPE/sentencepiece).

  CHARACTER: Too fine-grained. Devanagari matras (diacritics) are
    combining characters — "की" = क + ी (two code points). A character-
    level alignment would conflate encoding differences (NFC vs. NFD
    Unicode) with real errors, producing noisy edit counts.

  SUBWORD (BPE): Useful for language modelling but not for evaluation —
    the BPE vocabulary is model-dependent, so different models tokenise
    differently, making alignment across outputs inconsistent.

  WORD ← CHOSEN: Whitespace-delimited tokens after Unicode NFC
    normalisation and punctuation removal. Hindi is written in Devanagari
    with natural whitespace between words. Word-level WER is the industry
    standard (NIST, OpenASR). The Needleman-Wunsch alignment at word level
    directly maps to the S/D/I categories that WER measures. Compound
    split/merge cases are handled by placing both forms in the same bin.

1.3  LATTICE DESIGN
───────────────────
A lattice L for a segment of N reference words is a list of N sets (bins):

    L = [ Bin_0, Bin_1, ..., Bin_{N-1} ]

    Bin_i = { all valid word tokens at position i }

Construction steps:

  Step 1 — NORMALISE all transcriptions.
    Strip punctuation (। ? ! , . - —)
    Apply Unicode NFC
    Tokenise on whitespace

  Step 2 — ALIGN each model output to the reference.
    Use word-level Needleman-Wunsch to get an alignment of (ref_tok, hyp_tok)
    pairs. This tells us which model token corresponds to which reference
    position.

  Step 3 — SEED each bin with the reference token.
    Bin_i = { ref_tokens[i] }

  Step 4 — ADD all model tokens aligned to that position.
    This captures spelling variants, script variants, and filler words.
    Bin_i ∪= { hyp_tok aligned to position i from each model }

  Step 5 — MODEL-AGREEMENT OVERRIDE (trust models over noisy reference).
    If ≥ THRESH models (default 3 of 5) independently produced token T
    at position i, and T ≠ ref_tokens[i], we explicitly mark T as valid.
    (T was already added in Step 4; this step is the semantic justification
    for why models that produced T are not penalised.)

  Step 6 — OPTIONAL EPSILON BINS for consensus insertions.
    If ≥ THRESH models insert the same extra word at the same inter-word
    position, an optional epsilon bin { T, "ε" } is inserted. Models that
    include T match it for free; models that don't include T skip it for
    free (epsilon = zero cost).

1.4  WHEN TO TRUST MODELS OVER REFERENCE
──────────────────────────────────────────
Heuristic: If 3+ out of 5 models agree on a surface form that differs from
the reference, the reference is more likely wrong than all three models.
This threshold (majority of models) is conservative enough to avoid over-
accepting genuinely erroneous model outputs while capturing systematic
reference errors.

Examples from the data where this fires:
  • "नाते" (correct) vs reference "नाथ से" — 4 models agree
  • "हां" / "हाँ" — diacritic variation accepted across segments
  • "feedback" (Latin script) vs "फ़ीडबैक" — accepted as same token bin

1.5  WER AGAINST A LATTICE
────────────────────────────
    lattice_WER(L, H) = min edit distance(L, H) / |L_non_epsilon|

where the edit distance uses a modified cost:

    cost_match(h_token, Bin_i) = 0   if h_token ∈ Bin_i
                                = 1   otherwise

    cost_delete(Bin_i) = 0   if Bin_i contains "ε"  (optional position)
                       = 1   otherwise

    cost_insert = 1   (extra model token not in any bin)

This ensures:
  • Models that matched the reference are unaffected (token ∈ Bin_i → 0 cost).
  • Models that produced a valid variant are no longer penalised.
  • Models with genuine errors still pay the substitution/deletion cost.
  • WER can only decrease or stay the same; it never increases.

PART 2 — PSEUDOCODE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

──────────────────────────────────────────────────────
ALGORITHM 1: normalise(text)
──────────────────────────────────────────────────────
INPUT:  raw transcription string
OUTPUT: list of normalised word tokens

1. text ← unicode_NFC(text)
2. text ← strip_punctuation(text)          // remove । ? ! , . - — etc.
3. tokens ← split_on_whitespace(text)
4. RETURN [t for t in tokens if t ≠ ""]

──────────────────────────────────────────────────────
ALGORITHM 2: edit_align(ref_tokens, hyp_tokens)
──────────────────────────────────────────────────────
INPUT:  ref_tokens  = [r_0, r_1, ..., r_{R-1}]
        hyp_tokens  = [h_0, h_1, ..., h_{H-1}]
OUTPUT: alignment  = list of (r_i | None, h_j | None) pairs

1. Build (R+1)×(H+1) DP matrix dp[i][j] = min edits to align ref[:i] hyp[:j]
2. Initialise: dp[i][0] = i, dp[0][j] = j
3. FOR i = 1 to R:
     FOR j = 1 to H:
       match_cost = 0 if ref[i-1] == hyp[j-1] else 1
       dp[i][j] = min(
         dp[i-1][j-1] + match_cost,    // substitution / match
         dp[i-1][j]   + 1,             // deletion
         dp[i][j-1]   + 1              // insertion
       )
4. Traceback from (R, H) to (0, 0) to recover alignment pairs
5. RETURN alignment

──────────────────────────────────────────────────────
ALGORITHM 3: build_lattice(ref, model_outputs, thresh)
──────────────────────────────────────────────────────
INPUT:  ref           = normalise(reference)
        model_outputs = { model_name: normalise(transcript), ... }
        thresh        = 3  (MODEL_AGREE_THRESH)
OUTPUT: lattice = [Bin_0, Bin_1, ..., Bin_{R-1}]
                  plus optional epsilon bins for consensus insertions

1. FOR each model m:
     alignment[m] ← edit_align(ref, model_outputs[m])

2. FOR each model m:
     ref_pos ← 0
     FOR (r_tok, h_tok) in alignment[m]:
       IF r_tok ≠ None AND h_tok ≠ None:    // match or substitution
         ref_map[ref_pos][m].append(h_tok)
         ref_pos++
       ELSE IF r_tok ≠ None AND h_tok = None:  // deletion
         ref_map[ref_pos][m] = []
         ref_pos++
       ELSE:                                // insertion before ref_pos
         ins_map[ref_pos][m].append(h_tok)

3. lattice ← []
4. FOR p = 0 to R-1:
     bin_p ← { ref[p] }                    // seed with reference token

     // Step 4: add all model tokens aligned here
     FOR each model m:
       FOR each tok in ref_map[p][m]:
         bin_p.add(tok)

     // Step 5: MODEL-AGREEMENT OVERRIDE (explicit bookkeeping)
     counter ← Counter()
     FOR each model m:
       FOR each tok in ref_map[p][m]:
         IF tok ≠ ref[p]: counter[tok]++
     FOR tok, count in counter:
       IF count >= thresh: bin_p.add(tok)   // already added; annotate intent

     lattice.append(bin_p)

     // Step 6: consensus insertion epsilon bins
     ins_counter ← Counter all tokens in ins_map[p]
     FOR tok, count in ins_counter:
       IF count >= thresh:
         lattice.append({ tok, "ε" })       // optional position

5. RETURN lattice

──────────────────────────────────────────────────────
ALGORITHM 4: lattice_wer(lattice, hyp_tokens)
──────────────────────────────────────────────────────
INPUT:  lattice    = [Bin_0, ..., Bin_{L-1}]
        hyp_tokens = [h_0, ..., h_{H-1}]
OUTPUT: WER score

1. L ← len(lattice); H ← len(hyp_tokens)
2. IF L = 0: RETURN 0

3. Initialise (L+1)×(H+1) DP matrix
   dp[0][0] = 0
   FOR i = 1 to L:
     del_cost = 0 if "ε" ∈ lattice[i-1] else 1
     dp[i][0] = dp[i-1][0] + del_cost
   FOR j = 1 to H:
     dp[0][j] = j

4. FOR i = 1 to L:
     FOR j = 1 to H:
       h_tok ← hyp_tokens[j-1]
       is_optional ← ("ε" in lattice[i-1])

       sub_cost = 0 if h_tok ∈ lattice[i-1] else 1
       del_cost = 0 if is_optional else 1

       dp[i][j] = min(
         dp[i-1][j-1] + sub_cost,    // match / substitution
         dp[i-1][j]   + del_cost,    // deletion
         dp[i][j-1]   + 1            // insertion
       )

5. N_ref ← count of bins where "ε" ∉ bin   // non-optional reference words
6. RETURN dp[L][H] / N_ref

Case A — Script variant (Latin ↔ Devanagari)
  Segment: ch_487624  "जी फीडबैक मिलने पर सुधार करना"

  Segment: ch_452230  "आपका दिन शुभ हो धन्यवाद शुभ रात्रि"
  Model m : "आपका दिन शुब हो धन्यवाद शुब रात्री"
  "शुब"/"शुब" are phonetically correct spellings; 4 models wrote "शुभ"

Case D — Reference error (model consensus trumps reference)
  Segment: ch_687596  "तो एक भारतीय होने के नाते मुझे लगता है..."
  Human reference: "नाथ से" (clearly an error — "नाथ" means "Lord")
  Models i, k, l, m, n all write "नाते" (correct: "as being")
  → 5-model consensus overrides reference → "नाते" added to bin

  Segment: ch_1726922  "वही अपना खेती बाड़ी और क्या"
  Model m : "वही अपना खेतीबाड़ी और क्या"

                        │                       │ conservative enough threshold

                        │                       │ including nor omitting them
                        │                       │ should be penalised
